Reference Key Configuration

Reference Key: A unique identifier used for document matching in eligibility rules. For example, a disclosure code like "D164" that links documents to specific product arrangements.

Source APIs

Configure APIs that provide data for eligibility checks and document enrichment.

Why configure APIs? When checking eligibility (e.g., "only platinum members"), the system needs to know which API to call to get the customer's membership tier.

Credit Info API

Credit Card

Returns customer credit card information including membership tier, credit limit, and account status.

Endpoint: /api/v1/mock-api/credit-info
membership_tier credit_limit account_status card_type

Account Info API

Banking

Returns account details including balance, account type, and customer tenure.

Endpoint: /api/v1/mock-api/account-info
account_balance account_type customer_since state

Arrangements API

Product

Returns product arrangement details including pricing ID for disclosure matching.

Endpoint: /api/v1/mock-api/arrangements
pricing_id product_type disclosure_code

No API Required

Use this if eligibility is based on data already in the request (e.g., reference key matching).
